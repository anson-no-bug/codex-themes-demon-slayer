# 设计定义：鬼杀队任务中枢

## 方向选择

| 方向 | 主题对象 | 交互机制 | 相关性 | 交互潜力 | 可实现性 | 辨识度 | 合计 |
|---|---|---|---:|---:|---:|---:|---:|
| 鬼杀队任务中枢 | 讨伐令、鎹鸦、派遣小队 | 任务状态随受命→侦察→交战→归档连续变化 | 5 | 5 | 5 | 5 | 20 |
| 无限城工作台 | 无限房间、琵琶、门扉 | 分支任务像房间翻转与重排 | 4 | 5 | 2 | 5 | 16 |
| 蝶屋整备所 | 药柜、训练、伤情记录 | 设置与检查像整备和疗愈 | 3 | 3 | 5 | 4 | 15 |

推荐“鬼杀队任务中枢”。无限城的空间翻转会削弱 Codex 的导航方向感；蝶屋更适合设置页的局部语义。

## 两份契约

> 用**鎹鸦送达的讨伐令与小队作战状态**，让**Codex 从接受任务到验证完成的全过程**可感知，使用户感到自己在指挥一场有情报、有队员、有结果的行动，并自然地下达下一条任务。

视觉物理：以暖墨与炭黑作为地面，哑光和纸与旧金线为主要材料；紫藤提供低对比水印，绿色市松纹只用于窄栏边界，动画地点图作为景深。正文不放入毛玻璃或固定高度卡片，而由一层无边界、水平羽化的墨色阅读光场压低中央背景细节；文本保持锐利，原生操作控件不换名、不换图标。动效短促、克制，持续动画只用于明确的执行状态。

顶部任务条的对抗构图遵循“肖像在外、情报向内、冲突居中”：友方最多显示三张与恶鬼同高的纵向肖像，姓名最多三行；恶鬼肖像位于另一侧。友方原图顶部的环境留白通过对决区专属的 116% 主体裁切去除，不修改其它位置复用的素材。整个对决区只使用一条连续的阵营渐变：鬼杀队墨绿从左侧向中央衰减，鬼方暗红从右侧向中央衰减，在 `46%–54%` 合并为中性黑场；友方、鬼方与任务等级子区均为透明，不再形成拼接卡片。中央 `VS` 不使用徽章底板、容器边框或交叉斩痕，只用鬼杀队旧金、恶鬼赤色与明朝体笔锋建立阵营冲突。

### 层级与材质

1. **气氛层**：当前会话的官方动画地点图，仅承担空间与任务语义；正文、左栏与输入区分别使用适合自身比例的裁切，输入区只显示低对比局部场景。
2. **结构层**：暖墨面板、和纸颗粒、旧金分隔线、绿色市松收边与紫藤细节；它们建立层级，不承担主要信息。
3. **操作层**：任务标题、导航、输入文字、caret 与按钮沿用 Codex 原生语义；中央阅读光场只改变背景亮度，不占布局、不拦截输入。

## 交互映射

| Codex 对象/行为 | 鬼杀队叙事 | 反馈 |
|---|---|---|
| 新任务 | 接取讨伐 | 主入口显示“令”字受命印；项目内快捷接令保持常驻可辨，任务条与阶级印章落位 |
| 搜索 / Web 查询 | Codex 原生名称 | 保留原生展开、来源与搜索状态 |
| 输入框 | Codex Composer | 使用单一圆角、无阴影的地点场景底板，保留呼吸选择，不覆盖 placeholder 或操作标签 |
| 发送 / 停止 | 日轮发送状态 | 保持宿主按钮几何、标题、键盘与点击行为，以刀身 / 刀鞘和对应呼吸斩击表达发送状态 |
| 子代理 | 出战小队 | 队员席位与独立任务状态 |
| 计划 | Codex 原生名称 | 原生阶段与状态保持不变 |
| 终端 | Codex 原生名称 | 不再显示“锻刀工坊”等别名 |
| 文件 | Codex 原生名称 | 文件名与操作保持原样 |
| 右侧 Outputs | 任务案卷 | 保留原文件行与折叠事件，增加卷宗标记 |
| Environment / 分支 | Codex 原生名称 | 会话固定地点卡与原分支名并列 |
| Sources | 渡鸦情报 | 情报来源保持可展开、可点击 |
| Subagents | 出战小队 | 读取真实 background agents 并同步编成 |
| 测试 | Codex 原生名称 | 通过/失败均保留原生反馈 |
| 审批 | Codex 原生名称 | 保留原生批准与拒绝控件 |
| Token 余量 | 鬼王侵蚀 | 无惨 2D 人物固定站在组件侧边；十枚圆点离散表达侵蚀程度 |
| Token 已消耗 | 讨伐难度 | 消耗越深，任务从下弦侦察升级为鬼王终局 |
| 当前对手 | 同等级恶鬼 | 任务档位内随机抽取原著等级匹配的鬼，并在任务背板与悬浮条同步展示 |
| 当前分支 | Codex 原生名称 | 动画地点与原工程分支名同时保留 |
| 运行中 | 深夜追踪 / 交战中 | 呼吸环只在任务活跃时运转 |
| 完成 | 斩鬼归队 | 收刀线闭合，保留归档状态 |
| 失败 | 任务受阻 | 紫藤警戒纹与明确的再次追踪入口 |

## 原著人物、对手与 subagent 编成

| Token 已消耗 | 任务档位 | 对手池 |
|---:|---|---|
| 0–15% | 戊 · 下弦侦察 | 累 / 魇梦 |
| 15–35% | 丁 · 下弦讨伐 | 累 / 魇梦 |
| 35–55% | 乙 · 上弦末席 | 堕姬 / 妓夫太郎 / 狯岳 |
| 55–72% | 甲 · 上弦中位 | 玉壶 / 半天狗 / 鸣女 |
| 72–90% | 极 · 上弦前三 | 猗窝座 / 童磨 / 黑死牟 |
| 90–100% | 终 · 鬼王终局 | 鬼舞辻无惨 |

新会话从炭治郎、祢豆子、善逸、伊之助与完整九柱组成的 13 人正面角色池生成固定顺序。前两席优先从主角组抽取，再混入九柱；结果由稳定 thread ID 决定。React 重挂载和人数变化都不会整队重抽，人数增加只从既定顺序继续追加。

每个会话还会稳定取得一种任务类型：鬼迹追踪、失踪者搜救、紫藤护送、案卷审查、锻刀构建、分支合流、结界压测、渡鸦情报、依赖清剿、路由侦察、故障复盘或黎明交付。任务类型只改变任务条与案卷语义，不改写用户原始标题和操作。

左侧任务卡按真实“当前负责人 + background subagents”数量渲染：文字区右侧先留浅色雾化渐变，再将人物组织成 1、`x/x`、`x/x/x`、`x/x/x/+N` 的斜切肖像片；不使用圆形头像。斜线属于肖像内部的视觉分隔，不参与 flex 宽度计算，因此最后一名队员不会再被 `overflow:hidden` 裁掉。正文顶部将友方小队提升为与当前对手同级的独立模块，最多同时展示四名人物、姓名与职责。

## 会话固定地点

会话地点从十个动画场景中稳定抽取：

| 地点 | 任务语义 | 背景构图 |
|---|---|---|
| 无限城 · 深层回廊 | 高危潜入 / 终局 | 金色纵深，中央阅读罩加深 |
| 藤袭山 · 紫藤林 | 调查 / 试炼 | 紫藤满幅，人物偏左 |
| 吉原游郭 · 潜入夜街 | 多人侦察 / 追踪 | 主角三人居中，夜街暖光 |
| 蝶屋敷 · 修复庭院 | 修复 / 验证 / 归档 | 木屋与庭树，适合低压任务 |
| 浅草 · 夜市鬼迹 | 都市调查 / 搜救 | 大正夜市与暖色灯火 |
| 那田蜘蛛山 · 深林蛛网 | 依赖清剿 / 高压排障 | 冷绿深林与蛛网压迫感 |
| 刀匠村 · 云上工坊 | 构建 / 打包 / 交付 | 山谷云雾与隐村工坊 |
| 柱稽古场 · 瀑布修行 | 回归 / 压测 / 验收 | 山奥瀑布与高亮水雾 |
| 产屋敷宅邸 · 月夜庭院 | 审查 / 合流 / 复盘 | 月夜庭院与静态暗红 |
| 山顶废城 · 潜入侦察 | 路由追踪 / 故障定位 | 冷色废城与纵深走廊 |

地点不随 Token、难度或 DOM 重渲染变化；切换会话时读取另一份 thread state，切回时精确恢复。正文大背板、顶部任务地点、左侧任务栏、Composer 和右侧案卷消费同一个 `locationId`；Composer 以本地 `cover` 裁切读取对应地点，不使用固定背景。

十个地点为左侧窄栏提供独立焦点裁切，不把正文的宽幅构图直接压窄。左栏地点图位于深墨遮罩下，右缘以绿色市松纹和金线收束，紫藤只作为低对比细节；任务行与导航文字继续使用不透明墨层。输入区采用单一 22px 圆角裁切面，由两层深墨覆盖地点图；不使用 `box-shadow`、`filter` 或 `backdrop-filter`，编辑文字、placeholder、caret 和操作按钮均沿用原生语义。正文人物水印只使用友方编成，对手始终留在独立对手模块。

Token 状态不是普通额度条，而是反派压力层。真实 UI 有可访问余量信号时才显示比例；无惨透明全身 2D 人物保持固定尺寸和站位，进度只改变十枚圆点的亮灭。组件不显示 Token 换算说明与余量详情，并以固定宽度、`flex-shrink: 0` 保持几何稳定；窄屏时单独占据一行。没有可靠信号时保持未知态。

Composer 的承载面只恢复“任务地点”背景标识，不注入标题卡、装饰线、边框或阴影；标识在聚焦态仍保持可读，不再通过过低 opacity 退场。运行时只向上识别尺寸紧邻的 Composer 宿主包裹层并清除其背景渐变、投影和滤镜，不触碰正文或主布局。Composer 继续挂载随机 / 水 / 炎 / 雷 / 虫呼吸选择、日轮发送 / 收刀状态和对应斩击层。发送、停止、附件与语音按钮保持 Codex 原生几何、标题和事件；关闭“状态动效”时完整恢复原生发送图标与控件。

## 兼容原则

- 会话身份优先使用真实 `data-app-action-sidebar-thread-id`，不再把 React DOM 节点本身当作状态键。
- 左栏根节点从真实任务行向上取最近的 `aside`；composer 优先使用精确的原生根与表单结构，不用广谱 `aside` 或“最后一个按钮”推断目标，避免污染 Outputs 与其它工具面板。
- 小队人数只读取显式 `data-*`、可访问 subagent 节点和会话缓存；BigPizza 用户脚本不依赖 React 私有 fiber。
- 项目内新任务控件以 `[data-app-action-sidebar-project-row]` 为边界，只解除按钮自身 hover 外壳的透明状态；不展开相邻菜单，也不改变项目行的宽度、flex 或折叠行为。
- 原生任务行最右侧状态/归档轨道保持可点击并拥有最高操作层级；主题运行时测量其实际宽度，头像层只消费剩余空间。状态轨道出现时使用“两名头像 +N”的紧凑编成，避免把原生运行图标盖进人数徽记。
- 选择器失效时无操作回退，不拦截原按钮事件。
- Codex 宿主颜色 token 只存在于会话主容器、当前会话侧栏与摘要面板；非会话功能页和 Portal 浮层不接受主题标记或颜色覆盖。
- 所有替换文案在停用或重载用户脚本时还原。
- 主题关闭后移除根属性、样式、观察器、事件监听和注入节点。
- `prefers-reduced-motion` 下取消位移、旋转与循环，状态仍用文字和边框表达。
- 角色素材只用于本地学习构建，来源记录在 README；运行时离线加载，不进行远程请求。
